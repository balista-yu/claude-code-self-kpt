# 週次KPT自己振り返り

あなたはClaude Codeの自己改善エージェントです。今週の自分自身の作業を振り返り、KPTフレームワークで分析してください。

## 手順

### Step 1: 材料収集

以下の情報を収集してください

1. **今週の作業ログ**: `~/.claude/kpt-data/work-logs/` から今週分のファイルを読む
   - 現在のプロジェクトに絞る場合は、ファイル内の「プロジェクト:」行でフィルタ
   - 全プロジェクト横断で振り返る場合はすべて読む
   - ユーザーに「このプロジェクトのみ？全プロジェクト横断？」を最初に確認する
2. **Gitログ**: `git log --oneline --since="1 week ago" --until="now"` を実行（現在プロジェクト）
3. **前回のKPT結果**: `~/.claude/kpt-data/kpt/` から最新のKPTファイルを読む（あれば）
4. **現在のCLAUDE.md**: プロジェクトルートのCLAUDE.md と `~/.claude/CLAUDE.md` の両方を読む

作業ログが見つからない場合は、Gitログとdiffから可能な限り情報を復元してください。

### Step 2: 分析

収集した情報から以下を分析してください

- **繰り返し発生しているユーザー指摘**: 同じ種類の指摘が何回あったか数える
- **成功パターン**: ユーザー指摘なしで完了したタスクの共通点
- **失敗パターン**: 指摘が多かったタスクの共通点
- **前回Tryの実施状況**: 前回のKPTで挙げたTryが実際にできたか
- **プロジェクト横断の場合**: プロジェクト間で共通する課題パターンがないか

### Step 3: KPT生成

以下のフォーマットで `~/.claude/kpt-data/kpt/YYYY-WXX.md` に出力してください（YYYYは年、WXXは週番号）

```markdown
# 週次KPT: YYYY年 第XX週

## 分析期間
YYYY-MM-DD 〜 YYYY-MM-DD

## 分析対象
- スコープ: （このプロジェクトのみ / 全プロジェクト横断）
- 対象プロジェクト: project-a, project-b, ...

## 前回Tryの振り返り
| Try項目 | 実施状況 | 効果 |
|---------|---------|------|
| （前回のTry） | ○達成 / △部分的 / ×未達 | （効果の所感） |

## Keep（続けるべきこと）
### K1: （タイトル）
- プロジェクト: （どのプロジェクトで）
- 根拠: （なぜKeepなのか、データや事例を含む）
- 指摘なし率: XX%（該当タスクでの指摘なし率）

### K2: ...

## Problem（課題）
### P1: （タイトル）
- プロジェクト: （どのプロジェクトで）
- 発生回数: X回
- 具体例: （どのセッションでどんな指摘があったか）
- 根本原因の仮説: （なぜ繰り返すのか）

### P2: ...

## Try（次週やること）
### T1: （タイトル）
- 種別: hook / skill / CLAUDE.mdルール / ワークフロー
- 適用先: グローバル（~/.claude/） / プロジェクト（.claude/）
- 具体的アクション: （何をどう変えるか）
- 期待効果: （Problemの何が解決するか）
- 実装難易度: 低 / 中 / 高

### T2: ...

## 統計サマリー
- 総セッション数: X
- ユーザー指摘なし率: XX%
- 最頻出指摘: （内容とX回）
- 前週比: 指摘なし率 XX% → XX%（+X%）
```

### Step 4: 改善提案

KPTのTryについて、すぐに実装できるものがあれば提案してください。特に

- **hook化すべきもの**: 「CLAUDE.mdに書いても守れなかった」系のProblemはhookで自動チェック化
- **スキル化すべきもの**: 繰り返し成功しているパターンはスキルとして定義
- **CLAUDE.mdに追加すべきもの**: 本当にルールとして効果がありそうなものだけ厳選
- **適用スコープ**: 全プロジェクト共通の課題なら `~/.claude/` に、プロジェクト固有なら `.claude/` に

「TryをCLAUDE.mdに反映しますか？」とユーザーに確認してから反映してください。
